# 4.1 Prerequisites & File Deployment

Before configuring Nginx, you must securely transfer three specific assets generated by 
your automated PKI toolset to the target web instance:

1. **Server Private Key**: `srv-web01.local.key.pem`
2. **Full Chain Bundle**: `srv-web01.fullchain.pem` (Contains the Server Certificate + Intermediate CA Certificate)
3. **Root CA Certificate**: `ca.cert.pem` (Crucial for Nginx to locally validate OCSP signatures)

### Filesystem Security & Permissions

According to industry best practices, isolate these assets in a dedicated, secure directory 
structure with highly restrictive filesystem access control lists (ACLs):

```bash
sudo mkdir -p /etc/nginx/ssl

# Copy your files into this directory, then lock down permissions:
sudo chmod 600 /etc/nginx/ssl/srv-web01.local.key.pem
sudo chmod 644 /etc/nginx/ssl/srv-web01.fullchain.pem /etc/nginx/ssl/ca.cert.pem

# Establish root ownership to prevent non-privileged process tampering:
sudo chown -R root:root /etc/nginx/ssl/

```
